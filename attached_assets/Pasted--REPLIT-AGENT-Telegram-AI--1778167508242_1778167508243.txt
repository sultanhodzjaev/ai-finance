# МАСТЕР-ПРОМПТ ДЛЯ REPLIT AGENT
## Telegram-бот: AI-финансист для учёта личных трат

---

## КОНТЕКСТ ПРОЕКТА

Создай Telegram-бота на Python, который помогает пользователям вести учёт личных трат с помощью AI. Бот принимает траты в виде текстовых сообщений и фотографий чеков, автоматически категоризирует их через Gemini API, и работает как личный AI-финансист, отвечающий на вопросы о финансах пользователя.

**Целевая аудитория:** русскоязычные пользователи (СНГ, Кыргызстан, Россия).
**Язык интерфейса:** русский.

---

## ТЕХНИЧЕСКИЙ СТЕК

- **Язык:** Python 3.11+
- **Telegram-фреймворк:** aiogram 3.x (асинхронный)
- **AI:** Google Gemini API (модель `gemini-2.0-flash` для текста и Vision для фото)
- **Хранение данных:** Python-словарь в оперативной памяти (без БД и без файлов)
- **Конфигурация:** переменные окружения через Replit Secrets (`BOT_TOKEN`, `GEMINI_API_KEY`)

## ХРАНЕНИЕ ДАННЫХ

Все данные хранятся в **глобальном Python-словаре в памяти процесса**. Никаких файлов, никаких БД.

**Файл `services/storage.py`:**

```python
# Глобальный словарь — единственное хранилище данных
_storage = {
    "users": {}
}

def get_user(telegram_id: int) -> dict | None:
    return _storage["users"].get(telegram_id)

def create_user(telegram_id: int, username: str, first_name: str) -> dict:
    user = {
        "telegram_id": telegram_id,
        "username": username,
        "first_name": first_name,
        "currency": "KGS",
        "transactions": []
    }
    _storage["users"][telegram_id] = user
    return user

def add_transaction(telegram_id: int, transaction: dict) -> None:
    _storage["users"][telegram_id]["transactions"].append(transaction)

def get_transactions(telegram_id: int) -> list:
    user = get_user(telegram_id)
    return user["transactions"] if user else []
```

**Важно:**
- Данные **сбрасываются при перезапуске бота** — это нормально для текущего этапа
- НЕ создавай никаких JSON-файлов, не пиши на диск, не используй pickle
- Все операции синхронные — словарь работает мгновенно

## ФУНКЦИОНАЛ

### 1. Команда `/start`

Когда пользователь впервые запускает бота:
- Если его нет в хранилище — создай запись через `create_user()`
- Если уже есть — просто покажи меню
- Отправь приветственное сообщение:

```
👋 Привет, {first_name}!

Я — твой личный AI-финансист. Помогу тебе вести учёт трат без таблиц и заморочек.

🔥 Что я умею:
• 💬 Записываю траты по текстовому сообщению ("потратил 500 на обед")
• 📷 Распознаю чеки по фото
• 🤖 Отвечаю на вопросы о твоих финансах ("где я слил больше всего в этом месяце?")
• 📊 Показываю статистику за день, неделю, месяц

Просто напиши мне свою первую трату или нажми кнопку ниже 👇
```

С главным меню (inline-кнопки):
- 💬 Спросить финансиста
- 📊 Статистика
- ❓ Помощь

### 2. Запись трат текстом

Когда пользователь пишет любое текстовое сообщение (не команду):
1. Отправь его в Gemini API с промптом для категоризации (см. раздел "Промпты для Gemini")
2. Gemini возвращает JSON с полями: `amount`, `category`, `description`, `success`
3. Если AI не смог распознать (например, "привет") — отвечай: "Не понял, что ты хочешь записать. Напиши, например: 'потратил 500 на обед'"
4. Если распознал — покажи карточку подтверждения:

```
💰 Записать трату?

🍕 Категория: Еда
💵 Сумма: 500 сом
📝 Описание: обед

Сохранить?
```

С кнопками: ✅ Сохранить / ✏️ Изменить категорию / ❌ Отмена

При нажатии ✅ — добавь транзакцию в память через `add_transaction()`, отправь: "✅ Записал! Так держать."
При нажатии ✏️ — покажи список всех 10 категорий, дай выбрать другую, потом сохрани.
При нажатии ❌ — ничего не сохраняй, отправь: "Окей, отменил."

**Важно:** временные данные транзакции (до подтверждения) храни в FSM-state aiogram, не в глобальном хранилище.

### 3. Запись по фото чека

Когда пользователь отправляет фото:
1. Скачай фото через aiogram (получи bytes)
2. Отправь в Gemini Vision API с промптом для распознавания чека
3. Gemini возвращает JSON: `amount`, `category`, `merchant`, `description`, `success`
4. Если не удалось распознать — ответь: "Не получилось распознать чек. Попробуй сфотографировать ещё раз или введи трату текстом"
5. Если распознал — покажи карточку подтверждения (как для текста, но добавь поле "Магазин: {merchant}")
6. Логика подтверждения такая же как для текста

### 4. Команда `/ask` и кнопка "Спросить финансиста"

Когда пользователь нажимает кнопку или пишет `/ask`:
- Бот отвечает: "Задай мне любой вопрос про твои финансы. Например: 'сколько я трачу на еду?', 'где я слил больше всего в этом месяце?'"
- Включи FSM-state "ожидание вопроса финансисту"
- Следующее сообщение пользователя обрабатывается как вопрос к AI
- Через `get_transactions()` загрузи все траты пользователя из памяти
- Сформируй промпт с контекстом (см. раздел "Промпты для Gemini")
- Отправь в Gemini, получи ответ
- Верни ответ пользователю в Telegram
- Сбрось FSM-state

### 5. Команда `/today`

Покажи все траты пользователя за сегодня:

```
📅 Сегодня, 7 мая

🍕 Еда — 500 сом (обед)
🚗 Транспорт — 200 сом (такси)
🛒 Продукты — 1 200 сом

💰 Всего потрачено: 1 900 сом
```

Если трат нет — "Сегодня ты ещё ничего не записал. Так держать или забыл записать? 😄"

### 6. Команда `/stats`

Покажи общую статистику за текущий месяц:

```
📊 Статистика за май

🍕 Еда — 12 500 сом (45%)
🚗 Транспорт — 5 000 сом (18%)
🛒 Продукты — 4 200 сом (15%)
🎬 Развлечения — 3 800 сом (14%)
💊 Здоровье — 2 100 сом (8%)

💰 Всего: 27 600 сом
📈 Среднее в день: 920 сом
🔢 Транзакций: 34
```

Считай в коде на лету через фильтрацию `get_transactions()` по дате.

### 7. Команда `/help`

Краткая справка по командам:

```
📚 Команды:

/start — главное меню
/today — траты за сегодня
/stats — статистика за месяц
/ask — спросить AI-финансиста
/help — эта справка

💡 Просто напиши мне трату текстом или пришли фото чека — я сам всё распознаю!
```

## ОБРАБОТКА ОШИБОК

- **Gemini API не отвечает или возвращает ошибку:** "Что-то пошло не так с AI. Попробуй ещё раз через минуту"
- **Невалидный JSON от Gemini:** попробуй распарсить, при ошибке — сообщи пользователю что не смог распознать
- **Фото слишком большое или плохого качества:** "Не получилось обработать фото. Попробуй другое или введи текстом"
- **Все исключения логируй в консоль через `logging`** для дальнейшего дебага

## ВАЖНЫЕ ТРЕБОВАНИЯ

1. **Все секреты — через переменные окружения**:
   - `BOT_TOKEN` — токен от BotFather
   - `GEMINI_API_KEY` — ключ Gemini API
   - В Replit добавляй через Secrets, не хардкодь!

2. **Используй aiogram 3.x синтаксис** (не 2.x):
   - `Router` для регистрации хендлеров
   - `F` для фильтров
   - Async/await везде
   - FSM через `StatesGroup`

3. **Логирование:** настрой `logging` на уровне INFO

4. **Защита от ошибок Gemini:** оборачивай все запросы к Gemini в try/except

5. **Никакой записи на диск.** Все данные — только в Python-словаре в памяти.

6. **Polling, не webhook.** Webhook добавим в уроке деплоя.

7. **Время:** используй `datetime.now()` для меток транзакций.
